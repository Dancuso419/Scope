THE READOUT

Four checks and two stories

Every number is computed by rule-based code — same input, same result. The two narration blocks describe those numbers; they never produce them.

◆Concentrationhealth check · numeric
How much of the wallet's liquid value sits in a single token. Spam with fabricated prices is excluded first, so a fake “$5M” airdrop can't distort it.flags at ≥ 70% in one token
◆Stale approvalshealth check · security
Spending permissions you granted and forgot — unlimited, long-standing, still active, or flagged vulnerable. The quiet attack surface of an old wallet.unlimited + > 180 days + active, or vulnerable
◆Dead / illiquidhealth check · market
Held tokens whose on-chain market is too thin to exit cleanly — little liquidity or almost no recent trading.< $10k liquidity or < $100 / 24h volume
◆Value-only dusthealth check · numeric
Sub-$2 holdings, valued only. Always shown with a mandatory disclaimer: it ignores gas, liquidity, and spam/airdrop origin.holdings worth < $2
▹Health summarynarration · plain English
A short paragraph over the four checks, written for a human. The model narrates the facts it's given — it can't invent, estimate, or alter a figure.
▹Activity storynarration · plain English
A readable timeline of the wallet's notable trades and transfers over the recent window — what it did, in order, in words.
METHODOLOGY

The math is deterministic. The words are just words.

Scope keeps two layers strictly separate — that separation is the whole trust model.

Analysis layer RULE-BASED

All numbers and flags come from deterministic, non-LLM code reading live on-chain data. Reproducible and auditable — run it twice, get the same answer.

A liquidity gate filters out spam tokens before any math, so airdropped junk can't skew the results.

Narration layer GEMINI

The language model receives the already-computed facts, pre-rounded, and turns them into plain English — using the counts verbatim.

It cannot compute, estimate, or change a number, and never gives financial advice. If narration fails, the facts still return.

✓ no price predictions ✓ no buy / sell / hold language ✓ numbers never model-generated ✓ dust always carries its disclaimer ✓ read-only — no signing
