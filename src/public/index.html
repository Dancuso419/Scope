The readout

Four checks, two stories

Every figure is computed by rule-based code — same input, same result. The narration only describes those numbers; it never produces them.

01

Concentration

How much of the wallet's liquid value sits in one token. Spam with fabricated prices is filtered out first, so a fake “$5M” airdrop can't distort it.

flags at ≥ 70% in one token
02

Stale approvals

Spending permissions the wallet granted and forgot — unlimited, long-standing, still active, or flagged vulnerable. The quiet attack surface.

unlimited + > 180 days + active
03

Dead / illiquid

Held tokens whose on-chain market is too thin to exit cleanly — little liquidity or almost no recent trading.

< $10k liquidity or < $100 / 24h
04

Value-only dust

Sub-$2 holdings, valued only. Always shown with a mandatory disclaimer: it ignores gas, liquidity, and spam/airdrop origin.

holdings worth < $2

Methodology
Every number is computed by code. The model only writes the words. Nothing is invented, nothing is advised.

The analysis layer is deterministic, non-LLM code reading live on-chain data. Gemini receives the already-computed, pre-rounded facts and turns them into plain English — it can't alter a figure or give financial advice. If narration fails, the facts still return.

✓ no price predictions ✓ no buy / sell / hold language ✓ numbers never model-generated ✓ dust always carries its disclaimer ✓ read-only — no signing
